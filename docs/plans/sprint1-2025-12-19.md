# 주간/스프린트 계획 (Plan)

## 기간
- 시작: 2025-12-19
- 종료: 2026-01-02 (중간 점검: 2025-12-29)

## 목표
- 3개월 뒤 데모를 향한 1차 스프린트로, 모바일(Android/iOS) 전용 MVP 골격을 마련하고 FE/서버 로컬 실행 가능 상태 및 핵심 흐름(온보딩/이력/추천) 스켈레톤을 완성한다.

## 우선순위 작업
1. FE: 디자인 시스템 토대 + 온보딩/이력/추천 UI 스켈레톤, 비즈니스 로직 단위 테스트
2. 서버: CareerMap-Api 로컬 실행 + Auth/프로필/이력/추천 API 스켈레톤, 공통 에러 포맷, 비즈니스 로직 테스트
3. PM: 스레드/지시서 정렬, 중간 점검(12/29) 진행, 리스크/의존성 관리

## 역할별 할당
- PM: 스레드 관리, 중간 점검 진행, 리스크 트래킹 및 결정 기록
- 프론트엔드: 디자인 토큰/컴포넌트 1차 정의, 온보딩/이력/추천 UI 흐름 구현, 더미 데이터 기반 상태관리/검증 로직 테스트
- 서버: 로컬 환경 구성(H2/PG), Auth/프로필/이력/추천 기본 API 스켈레톤, 에러 포맷/예외 핸들러, 서비스/도메인 테스트
- 문서화: 로컬 실행/테스트 절차 정리, 스레드/결정 기록 최신화

## 리스크 및 대응
- 서버 API 지연 → FE는 더미 데이터/스텁으로 병렬 진행, API 계약은 ERD 기준으로 합의 후 스레드 기록.
- 디자인 자산 부재 → FE가 전권 설계, 토큰/컴포넌트 초안 공유 후 빠른 피드백.
- 일정 변동(연휴/가용 리소스) → 중간 점검에서 범위 재조정, P1 기능 후순위화.

## 완료 기준
- FE: 온보딩/이력/추천 주요 화면 스켈레톤 동작, 디자인 토큰/핵심 컴포넌트 정의, 비즈니스 로직 단위 테스트 통과.
- 서버: 로컬에서 핵심 API 스켈레톤 및 헬스체크 동작, 공통 에러 포맷 적용, 서비스/도메인 테스트 통과.
- PM/문서: 스레드에 진행/차단/다음 액션 기록, 결정/리스크 문서화, 중간 점검 결과 정리.
